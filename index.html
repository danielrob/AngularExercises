Angular Forms

1. <form name="AngularForm"> - this associates the form with an object which is seen below. You can change the submitted status in this example. 


{{AngularForm | json }}

2. Add <input ng-model="something"> - here the difference is the input has an ng-model directive. Note that you can now make pristine change in this example. 


{{AngularForm1 | json }}

3. Add < input name="myShortStory" ng-model=""> - here the difference is the input has a name as well as ng-model. Now the form has a 'myShortStory' attribute. 


{{AngularForm2 | json }}

4. Add < form ng-submit="myVar='intercepted!'"> - Using ng-submit will stop the browser from sending an HTTP request and evaluate the specified expression instead.

I have been: {{myVar}}

5. You can use ng-click instead of ng-submit - The same thing can be achieved using ng-click on the first input/button with type submit in the form.

I have been: {{tyd8o}}

Angular Form Data

From the above, you can see that using just directives, you can fill a form object and populate the scope with either/both a form object, and/or ng-model model data. Logic on this data can be performed in the view, or if the form is wrapped in a controller, it follows you could call to functions on the controller, and have it act on form data.

Without a controller, the form object can be used to customise the user experience of the form using other directives like ng-show or ng-if.

<div ng-show="form.$submitted">
  <p>Form Submitted!</p>
</div>

This is useful in particular for validation. For example if a field is required, and it has not yet been completed, it will have the following property:

formName.fieldName.$error.required

- This is explored in more detail below.

Angular also includes CSS classes based on the form and field states. See the angular docs for details.

Non-display logic should be placed in the controller.

Angular Form Validation

As seen in the previous section when we use the name directive on a form, and use named form fields, we can see the form and field states using the form object. In addition we can populate the field properties with further validation properties, to give us full control over form state. First though, we want to disable HTML5 Form Validation.

1a. <form name="aForm"> - Firstly, a form like before: 

Submitted...................................: {{ngValidationForm.$submitted}}

1b. <form name="aForm" novalidate> - Now, same form but with novalidate. 

Submitted: {{ngValidationForm0.$submitted}}

So.... you can use novalidate to disable native HTML form validation.
